# 🍠 XHS Quad Brain - 小红书四脑协作系统

四角色协作生产小红书爆款内容：内容策划 → 文案创作 → 算法审查(失败重写) → 资产沉淀

## 何时使用

- 创作小红书爆款笔记
- 分析/复刻对标账号内容
- 优化现有文案去AI化
- 批量生产矩阵号内容
- 沉淀可复用的内容SOP

## 快速使用

直接告诉我要用四脑协作创作小红书内容：

> "用四脑协作写一篇关于早八妆的小红书"

我会自动运行完整流程。

## 四脑角色定义

### 📝 PM·内容策划
**职责**：选题分析、受众定位、竞品对标

**输出**：《选题策略书》
- 爆款选题池（3-5个方向）
- 目标人群画像（痛点/爽点/场景）
- 竞品对标分析
- 差异化定位建议

### 💻 DEV·内容创作  
**职责**：文案撰写、视觉策划、SEO优化

**输出**：《内容成品包》
- 标题（3个备选）
- 正文文案（钩子→痛点→干货→CTA）
- 封面描述（Live/大字报/拼图方案）
- 标签矩阵（核心词+场景词）

### 🔍 REVIEWER·算法审计
**职责**：AI味检测、合规审查、爆款潜力评估

**审查维度**：
- 🚫 **去AI化**：排比句/说教感/华丽词藻（AIGC毒性检测）
- ⚖️ **合规性**：违禁词/诱导私信/广告法
- 📈 **爆款潜力**：完读率锁/互动诱饵/私域钩子
- 🎭 **人设一致性**：是否符合账号调性

**输出**：`VERDICT: PASS` 或 `VERDICT: FAIL` + 具体问题清单

### 📋 MEMO·资产沉淀
**职责**：生成可复用的内容资产

**输出**：
- 《内容生产SOP》- 成功经验提炼
- 《爆款模板库》- 可套用的标题/结构公式
- 《避雷清单》- 本次踩过的坑
- JSON资产包 - 可导入自动化工具

## 工作流程

```
用户输入需求/对标内容
    ↓
📝 PM - 分析选题策略
    ↓
💻 DEV - 创作初版内容
    ↓
🔍 REVIEWER - 审查内容质量
    ├─ FAIL → 反馈问题给DEV重写（循环，最多max_retries次）
    └─ PASS → 继续
    ↓
📋 MEMO - 生成资产沉淀
    ↓
输出完整报告 + 可复用资产
```

## XHS-Deep-Decoder v5.0 核心逻辑

### 2026 CES 算法要点
- **含人量 (Humanity Score)**: 打击AI味，推崇瑕疵感与情绪宣泄
- **私域价值**: 识别群聊引导、商卡转化
- **SEO场景化**: 提取生活场景词（Context Tags）

### AIGC 毒性检测
高风险词汇：`首先/其次/致力于/非常/极其/因此`

修正方向：`口语化/情绪化/带个人色彩`

### 爆款结构公式
```
0-3秒钩子: 情绪崩溃/巨大反差/直击痛点
中段: 真实经历 + 干货植入（过载信息促收藏）
结尾: CTA引导（争议提问/求资料/进群）
```

## 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| max_retries | 最大重写次数 | 2 |
| format | 输出格式 | graphic(图文)/video(视频) |
| humanity_target | 目标含人量 | 80+ |

## 使用场景示例

### 场景1：新品推广
> "用四脑协作写一篇关于XX面膜的小红书推广"

PM分析护肤赛道 → DEV创作种草文案 → REVIEWER检查硬广味 → MEMO沉淀模板

### 场景2：人设账号
> "用四脑协作写职场博主的内容，目标人群是95后打工人"

PM定位人设 → DEV写职场吐槽 → REVIEWER检查真实感 → MEMO积累选题库

### 场景3：矩阵号批量生产
> "用四脑协作批量生成10篇家居好物内容"

PM规划选题矩阵 → DEV批量创作 → REVIEWER统一风格 → MEMO输出标准化流程

## 注意事项

1. **迭代次数**：建议 max_retries=2，过多迭代可能偏离初衷
2. **人设一致性**：告诉PM账号定位，避免REVIEWER阶段反复调整
3. **合规优先**：REVIEWER会严格检查，宁可FAIL也不冒险
4. **资产复用**：每次MEMO输出的模板要保存，下次可加速生产
