# 京东消息重复发送Bug - 严重状态污染

## Bug描述
用户在QQ上收到了4次未经请求的京东配送消息（昨晚20:45、凌晨0:00、3:00、4:30）。

## 关键发现
1. 用户确认没有在QQ或Discord上询问京东
2. Discord session历史显示，在cron任务执行时间点，我**自动**发送了京东回复
3. 这些回复与cron任务通知同时出现

## 根本原因推测
**Session状态污染** — 某个之前的京东查询残留在我的上下文中，导致：
- 每次处理cron任务system message时
- 我的thinking中包含京东配送内容
- 错误地将京东回复作为对cron任务的响应

## 证据
Discord session历史显示：
```
[2026-02-15 00:00:38] 我回复了京东配送（用户未发消息）
[2026-02-15 03:00:29] 我回复了京东配送（用户未发消息）
[2026-02-15 04:30:15] 我回复了京东配送（用户未发消息）
```

## 严重性
- **高** — 自动发送未经请求的消息
- **隐私风险** — 可能泄露其他用户的查询内容
- **用户体验** — 重复骚扰用户

## 建议修复
1. 清除所有session的京东相关上下文
2. 检查cron任务system message处理逻辑
3. 添加回复内容验证，确保与当前查询相关
4. 实施消息去重机制

## 状态
等待OpenClaw团队修复
